What's a promise?

A promise is an object that contains two properties (state and result). The state will always be one of the following three: pending, resolved, rejected. A promise will remain in pending until it receives a result. The result will always be either resolved (the result comes back successfully) or rejected (there was an error with getting the result). The result will change depending on where/how you use the promise. Oftentimes, a promise is used with API calls, because promises allow for asynchronous coding. This allows you to make an API request and the rest of your code will continue, but your promise will promise> to execute something depending on whether your request was made successfully or will execute something else if there was an error with the request.

Three States
  • pending: your promise is still waiting for a response/result
  • resolved: your promise may continue to execute because the response/result was successful
  • rejected: your promise will not continue because the response/result was not successful, there was an error

Basic Fetch Get Request

fetch('url')
  .then(response => response.json())
  .then(data => console.log(data));
let's break it down:

.fetch('url'): this takes in the url which you're wanting to make the call

.then(): this returns a promise and is chain-able and takes in a function whose parameter is the result of a promise (from fetch or previous then statement) and this will be executed if the promise is SUCCESSFUL

.catch(): this returns a promise and is chain-able and takes in a function whose parameter is the result of a promise (from fetch or previous then statement) and this will be executed if there is an ERROR

Let's add .catch()

fetch('https://jsonplaceholder.tyicode.com/todo')
    .then(response => response.json())
    .then(json => console.log(json))
    .catch((error) => {
    console.error('Error caught in catch method:', error);
});
let's break it down:

.catch(): this will catch and return any errors, also chain-able

Fetch using POST request

const data = { title: 'example-title' };

fetch('https://jsonplaceholder.typicode.com/todos', {
    method: 'POST', // or 'PUT'
    headers: {
        'Content-Type': 'application/json',
    },
    body: JSON.stringify(data),
})
    .then(response => response.json())
    .then(data => {
        console.log('Success:', data);
    })
    .catch((error) => {
        console.error('Error:', error);
    });
let's break it down:

.fetch('url', {options}): this takes in the url which you're wanting to make the call
{options}: To do a POST request we need to specify additional parameters with the request such as method, headers, etc. 

.fetch('url', {
    // specify which type of request you're making
            method: 'POST',
            headers: {},
    // where you pass in actual data being added
            body: JSON.stringify(data)
        })
: options contains all the extra info needed in order to make a different kind of request and to send more info when making a request
